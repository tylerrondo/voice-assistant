# Platform Vision

## Назначение
Объясняет зачем существует платформа, а не как она устроена.

## Исходная проблема
Каждый интерфейс (Web, Mobile, Voice, Map, WhatsApp) развивается независимо
и создает собственную инфраструктуру. Платформа решает эту проблему.

## Идея платформы
Все интерфейсы - это каналы взаимодействия с одной системой.
Голос, карта, чат - не особые случаи. Все они каналы.

## Единая модель
Пользователь -> Канал -> InteractionAction -> InteractionContract -> InteractionEvent -> Канал -> Пользователь

## Что переиспользуется
Interaction Contract, Scenario Engine, Execution Log, Verification Harness, Validation Bench.

## Стратегическая цель
Добавление нового интерфейса сводится к реализации Provider, Mapper, Channel.
Всё остальное уже существует.

## Следующий этап
Миграция Map-модуля на Platform Core - первое доказательство универсальности платформы.
